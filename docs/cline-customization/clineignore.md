### .clineignore 支持

为了给您更多控制权来决定 Cline 可以访问哪些文件，我们实现了类似 `.gitignore` 的 `.clineignore` 功能。这允许您指定 Cline **不应**访问或处理的文件和目录。这在以下场景中非常有用：

*   **隐私保护**：防止 Cline 访问工作区中的敏感或私有文件
*   **性能优化**：排除与任务无关的大型目录或文件，可能提升 Cline 的运行效率
*   **上下文管理**：将 Cline 的注意力集中在项目的相关部分

**如何使用 `.clineignore`**

1.  **创建 `.clineignore` 文件**：在您工作区的根目录（与 `.vscode` 文件夹同级，或您在 VS Code 中打开的顶级文件夹）中创建一个名为 `.clineignore` 的新文件

2.  **定义忽略规则**：打开 `.clineignore` 文件并指定您希望 Cline 忽略的文件和目录模式。语法与 `.gitignore` 相同：

    *   文件中的每一行代表一个模式
    *   **支持标准 glob 模式**：
        *   `*` 匹配零个或多个字符
        *   `?` 匹配一个字符
        *   `[]` 匹配字符范围
        *   `**` 匹配任意数量的目录和子目录

    *   **目录模式**：在模式末尾添加 `/` 来指定目录
    *   **否定模式**：在模式前加 `!` 来取消之前忽略的模式
    *   **注释**：以 `#` 开头的行会作为注释

    **示例 `.clineignore` 文件**：

    ```
    # 忽略日志文件
    *.log

    # 忽略整个 'node_modules' 目录
    node_modules/

    # 忽略 'temp' 目录及其子目录中的所有文件
    temp/**

    # 但不要忽略根目录下的 'important.log' 文件
    !important.log

    # 忽略任何子目录中名为 'secret.txt' 的文件
    **/secret.txt
    ```

3.  **Cline 会遵守您的 `.clineignore`**：保存 `.clineignore` 文件后，Cline 会自动识别并应用这些规则

    *   **文件访问控制**：Cline 将无法使用 `read_file` 等工具读取被忽略文件的内容。如果您尝试对被忽略文件使用工具，Cline 会告知您由于 `.clineignore` 设置导致访问被阻止
    *   **文件列表**：当您要求 Cline 列出目录中的文件（例如使用 `list_files` 时），被忽略的文件和目录仍会被列出，但会在名称旁显示 **🔒** 符号，表示这些内容已被忽略。这有助于您了解 Cline 可以与哪些文件交互

4.  **动态更新**：Cline 会监控您的 `.clineignore` 文件变更。如果您修改、创建或删除 `.clineignore` 文件，Cline 会自动更新忽略规则，无需重启 VS Code 或扩展

**总结**

`.clineignore` 文件提供了一种强大而灵活的方式来控制 Cline 对工作区文件的访问，增强了隐私保护、性能优化和上下文管理能力。通过利用熟悉的 `.gitignore` 语法，您可以轻松地将 Cline 的注意力集中在项目中最相关的部分。
